# ACM SIGSOFT Empirical Standards — レビュー用チェックリスト

Source: https://github.com/acmsigsoft/EmpiricalStandards
Paper: Ralph et al. "Empirical Standards for Software Engineering Research" (2020)

## 研究手法の判定

論文を読み、以下のいずれの手法に該当するか判定する:

| 手法 | 判定基準 |
|------|---------|
| Experiment | 独立変数の操作、仮説検定、実験群/対照群の比較 |
| Case Study | 実世界の文脈での現象の深い調査、少数の事例 |
| Repository Mining | ソフトウェアリポジトリ（GitHub, StackOverflow等）からのデータ収集・分析 |
| Engineering Research | ツール・アルゴリズム・フレームワーク等の技術的成果物の構築と評価 |
| Benchmarking | 標準化されたベンチマークによるシステム性能評価 |
| Questionnaire Survey | 質問票による体系的調査 |
| Systematic Review | 既存研究の体系的収集・統合 |
| Mixed Methods | 複数の手法の組み合わせ |

複数手法を用いている場合は、主要手法と補助手法をそれぞれ判定する。

## General Standard（全研究共通）

### Essential（必須）
- [ ] 目的・問題・研究課題を明記している
- [ ] なぜ重要かを説明している（動機）
- [ ] 専門用語・略語・主要概念を定義している
- [ ] 使用する研究手法を明示している
- [ ] 手法が目的に対して適切である
- [ ] データ収集の what/where/when/how を詳述している
- [ ] データ分析方法を詳述している
- [ ] 結果を提示している
- [ ] 結果が研究課題に直接対応している
- [ ] 統計検定の仮定を列挙・検証している（該当する場合）
- [ ] 結果の含意を議論している
- [ ] 主要な制約を開示している
- [ ] 結論が研究課題と結び付き、根拠に裏付けられている
- [ ] 知識体系への貢献がある
- [ ] 読みやすい文章で書かれている

### Desirable（望ましい）
- [ ] 認識論的立場を述べている
- [ ] 関連研究を適切に統合している
- [ ] 貢献と関連研究の関係を明確にしている
- [ ] 統計的検定力 or 質的飽和を示している
- [ ] 制約の調査・軽減を試みている
- [ ] 実践的な解釈・推奨を提供している
- [ ] 簡潔で整理された記述
- [ ] 主観的判断に複数評価者を使用している

## Experiment（実験）

### Essential
- [ ] 形式的な仮説を述べている
- [ ] 両側仮説を使用、または片側の正当化がある
- [ ] 従属変数を記述し、測定方法を正当化している
- [ ] 独立変数の操作/測定方法を記述している
- [ ] 外生変数とその制御方法を記述している
- [ ] 標本サイズを正当化している（例: 検定力分析）
- [ ] 研究デザインとプロトコルを記述している
- [ ] ランダム割当て、または準実験の正当化がある
- [ ] 実験対象を記述し、選定を正当化している
- [ ] 参加者の属性を記述している
- [ ] 分布に適した記述・推測統計を報告、使用した検定を正当化
- [ ] 効果量と信頼区間を報告している
- [ ] 構成概念・結論・内部・外部妥当性を議論している
- [ ] 結果の代替解釈を議論している

### Desirable
- [ ] 補足資料（プロトコル、材料、データセット、分析スクリプト）
- [ ] 代替実験デザインとその不採用理由を議論
- [ ] データ分布の可視化
- [ ] 仮説・デザインの事前登録

---

## Case Study（ケーススタディ）

### Essential
- [ ] 事例・サイトの選定を正当化している
- [ ] サイトを詳細に記述している
- [ ] サイトは実際の場所（トイ例ではない）
- [ ] ケーススタディの種類を報告している
- [ ] データソースを記述している
- [ ] 分析/観察の単位を定義している
- [ ] 観察から発見への明確な証拠の連鎖を提示している

### Desirable
- [ ] 補足資料（インタビューガイド、コーディング例等）
- [ ] データソース・情報提供者・研究者間の三角測量
- [ ] メンバーチェック等による結果の検証
- [ ] 引用による発見の例示
- [ ] 研究者自身のバイアスへの省察

---

## Repository Mining（リポジトリマイニング）

### Essential
- [ ] リポジトリマイニングが研究問題に適する理由を説明
- [ ] 分析/観察の単位を定義している
- [ ] データソースを記述・正当化している
- [ ] リポジトリの選定方法を記述・正当化している
- [ ] 包含/除外基準の検証方法を記述している
- [ ] 選定されたリポジトリを記述している
- [ ] データ取得手順を記述している
- [ ] データセットの特性（サイズ、属性）を記述している
- [ ] データ前処理を記述している
- [ ] 手動アノテーション時: 複数アノテーター、評価者間信頼性
- [ ] 使用する指標を記述・正当化している
- [ ] 外部妥当性への脅威を議論している

### Desirable
- [ ] 補足資料（データセット、ツール）
- [ ] 確率的サンプリング
- [ ] 構成概念妥当性の定量的評価
- [ ] ツールの単体テスト
- [ ] リポジトリマイニングの倫理的問題の議論

---

## Engineering Research（工学的研究）

### Essential
- [ ] 提案する成果物を十分な詳細で記述している
- [ ] 成果物の必要性・有用性を正当化している
- [ ] 成果物を概念的に評価している（強み・弱み・限界）
- [ ] 経験的評価方法を明示している
- [ ] 最先端の代替手法と比較、またはその不在/比較不能の正当化
- [ ] ベンチマーク/代替手法との比較、またはその省略の正当化
- [ ] 仮定が明示的で妥当

### Desirable
- [ ] 再現パッケージ（ソースコード、データセット）
- [ ] 理論的基盤の議論
- [ ] 動作例の提示
- [ ] 産業的に現実的な文脈での評価
