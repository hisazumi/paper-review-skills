# Student Comment — 教員向けレビューを学生向け1枚に翻訳

教員向けの厳しい分析（`research-review` などの統合レビュー`.md`）を入力に、**学生に直接渡せる中間発表コメント1枚**へ書き換える。分析はしない（それは上流の担当）。ここは**トーンと構成の翻訳層**。

## When to Use

- レビュー結果を「学生向けに書き換えて」「学生に渡せる形に」と言われたとき
- `research-review <file> --student` から連鎖呼び出し

## Input

$ARGUMENTS

教員向けレビュー`.md`（1本）を読み、その指摘を漏らさず学生向けに再構成する。原稿本体(.tex)は編集しない（**本文執筆は学生の仕事**）。

## 規範（厳守）

- **トーン**: 前向きだけど抑えめ・シンプル。冒頭で土台を一言だけ認める。過剰な褒めはしない。厳しさは事実で示す。
- **禁句**: 「（今回は書かなくても減点されません）」のような"減点しない"系の但し書きは**使わない**（上から目線・保険的に響く）。「提出後でよい」で止める。
- **self-contained**: 別ファイルを開かなくても手が動くよう、**例文・下書きを本文にインライン**で載せる。外部参照に依存しない。
- **例文の扱い**: 「そのまま写すな、自分の言葉に直せ」と必ず添える。
- **優先順**: 要確認（学生が決める土台があれば最上位）→ 提出まで（この順で）→ できれば → 提出後 → 体裁。
- **理由の逃がし方**: 詳しい背景は末尾「補足（読まなくても手が動く）」へ。本体はまっすぐ読める行動リストに保つ。
- **框の付け替えを含むとき**: 冒頭に「中間では看板の1〜2文差し替えだけでよい／実装は今のまま」の**安心材料**を置き、学生がパニックにならないようにする。

## Output

`<元レビュー名>-学生向けコメント-<YYYY-MM-DD>.md`（`--out` 指定時はそこへ）。

```markdown
# 中間発表コメント（[氏]くんへ）
[YYYY-MM-DD] ／ 久住

[土台を一言認める＋（框の付け替え時は）安心材料。詳しい理由は末尾の補足、と一言]

## 要確認（最優先）：… ← 学生が決める土台があるときだけ
[選択肢と、教員の見立て。決めないと後続がやり直しになる旨]

## 提出まで（この順で）
**1. …** [何を・どこに・なぜ]
> 例）[インライン下書き]
**2. …**
**3. …**

## できれば（余裕があれば、1文ずつ）
- …

## 提出後でよい
- …

## 体裁（Minor・余裕があれば）
- …

---

## 補足：…（読まなくても手は動きます）
[なぜこの順か・厳しい所の背景を、学生が飲み込める粒度で]
```

## Guidelines

- 教員レビューの指摘を**取りこぼさない**が、学生の締切負荷を考えて「提出まで／できれば／提出後」に**負荷配分**する。
- Critical/Major は「提出まで」、長尺前提は「提出後」へ。
- 分析の蒸し返し・新規の弱点発掘はしない。翻訳と優先順位付けに徹する。
